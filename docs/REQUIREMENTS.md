# 需求规格文档

## 项目目标
构建一款 Windows x64 桌面应用，基于古诗词通过AI生成短视频，并提供视频管理功能。

## 用户角色
- 内容创作者：使用工具将古诗词转化为短视频内容

## 功能需求

### FR-1 古诗词选题
- 每次随机展示 4 首未使用过的古诗词（绝句/律诗/词）
- 展示内容：标题、作者、朝代、全文
- 用户选择一首作为创作题材
- 支持"换一批"重新随机
- 去重：已使用的诗词不再出现

### FR-2 AI场景描绘
- 调用豆包API，将古诗词用现代语言描绘成视觉场景
- 用户可修改描绘内容
- 修改后确认进入下一步

### FR-3 AI视频提示词生成
- 调用豆包API，基于场景描述生成结构化视频提示词
- 固定约束：
  - 视频时长 ≤ 1分钟
  - 人物形象：带有可爱猫咪特征的人
  - 总体风格：卡通可爱
- 输出格式（结构化）：
  ```
  【主体】[年龄/性别/特征/服装] 的 [主体名称]
  【细节】[特有标志，如发型、道具、配饰]
  【动作】[具体动态，包含速度、幅度、方向]
  【场景】[地点 + 时间 + 周围关键元素]
  【镜头】[景别] + [运镜方式] + [角度]
  【光影】[光源位置/颜色/软硬]
  【氛围】[情绪词，如紧张、梦幻、史诗]
  【画质】[分辨率] + [风格流派] + [参考作品名]
  ```
- 用户可逐项编辑各字段

### FR-4 AI视频生成
- 调用即梦API（Doubao-Seedance-2.0）生成视频
- 异步任务，轮询状态直至完成
- 下载视频到本地临时目录
- 展示视频预览（内嵌播放器）
- 用户选择"保存"或"放弃"

### FR-5 视频管理
- 视频文件保存到用户指定的本地文件夹
- 元数据记录：诗词信息、场景描述、提示词、生成时间、文件信息
- 功能：
  - 视频列表浏览（缩略图 + 信息）
  - 搜索/筛选
  - 播放预览
  - 删除（同时清理文件和元数据）

## 非功能需求

### NFR-1 UI/UX
- 简洁直观的界面设计
- 主色调：淡蓝色系
- 步骤式创作流程，带进度指示器
- 良好的操作反馈（loading、成功、错误状态）

### NFR-2 性能
- 应用启动时间 < 3秒
- 界面操作响应 < 200ms

### NFR-3 安全
- API Key 通过 .env 管理，不硬编码
- 本地数据存储在用户目录下

### NFR-4 平台
- 操作系统：Windows 10/11 x64
- 打包格式：exe 安装包
